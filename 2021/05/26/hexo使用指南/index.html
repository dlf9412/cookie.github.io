hexo使用指南

发表于2021-05-26|更新于2023-05-26|hexo
|阅读量:

hexo安装

npm intall -g hexo

建站

新建一个hexo目录，cd进入目录，执行命令行
hexo init
执行完成会看到生成的结构

本地服务启动

  • 首先先检查目录中是否有public 文件夹，且文件夹中是否有index.html文件，因为本地服务访问的是public的文件，所以服务启动必须要有这个index.html，否则就算正常启动了，也会报404。
  • 如果检查没有这个结构，则执行如下命令行生成
    hexo generate或hexo g,后面这个为简写
  • 如果生成的public文件夹中没有index.html，检查package.json 的依赖是否有hexo-renderer-marked这个依赖，如果没有先安装依赖，再执行
    npm install hexo-renderer-marked --save
  • 检查无误，执行命令行，启动服务
    hexo server或hexo s
  • 默认启动的端口为4000，如果端口被占用
    hexo serve -p 5000

hexo文章生成

  • 初始化完成后会默认生成hello-word.md，如果想要生成我们自己的文章，可以通过hexo new "文件名"
  • 生成的文件会source/_posts中找到

hexo上传

  • 编写完文章以后想要上传到github 上
  • 首先要进行配置，找到_config.yml文件中的deploy配置项,复制github中的地址，进行如下配置
  • 清理旧数据
    hexo clean
  • 生成文件
    hexo g或者hexo generate
  • 配置完成，上传文件
    hexo d或者hexo deploy

hexo主题使用

hexo图片显示

图片显示问题可以参考这篇文章：https://zhuanlan.zhihu.com/p/542101567

hexo隐藏文章

  • 安装依赖
    npm install hexo-hide-posts --save
  • 修改_config.yml配置项
  • 在文章的属性中定义secret:true即可隐藏文章

hexo 置顶文章

  • 安装依赖
    npm install hexo-generator-index-custom
  • 配置
  • 使用方式
    在文章开头配置中添加sticky或top 参数，值为true或数字，数字值越大，文章排越前

hexo文章分类

  • 创建分类选项
  • 生成“分类”页并添加type 属性，进入博客目录。执行命令下方命令
    hexo new page categories
  • 会在source/categories下生成一个index.md，打开后得默认内容
  • 添加type: categories到内容中
  • 打开要分类的文章，给文章的font-formatter中添加categories属性，可以添加一个或者多个
  • 只要设置了categories: helloword的文章，都会被划分为一类，也可以设置categories为其他值
  • 添加了categories: xxx 会被收录到首页的“分类”中

hexo文章标签

  • 生成标签页
    hexo new page tags
  • 生成的文件在source/tags的index.md,打开默认内容为
  • 添加type:’tags’ 到内容中
  • 给文章添加’tags’属性，打开需要添加标签的文章,可以添加一个或者多个标签

配置项填写注意事项

_config.yml的配置项填写的时候要注意间距，冒号后面需要空一格，否则无效
